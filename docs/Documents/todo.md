### բնակարանների ցուցադրում նոր ձեւաչափով ( ըստ հարկերի , հատակագծով ).

## 📌 Ֆունկցիոնալ 1 — Անհատական լենդինգի ավտոմատ գեներացում

## (քարտից)

```
 Լենդինգի ստեղծում եկ կոճակով
Բնակարանի վաճառքից հետո CRM-ում համապատասխան կոճակը սեղելու իջոցով ավտոմատ
ստեղծվում է լենդինգ՝ տվալ բնակարանի քարտի հիման րա։
```
```
 Քարտի տվալների ավտոմատ փոխանցում
Լենդինգում ավտոմատ արտացոլվում են բնակարանի բոլոր տվալները՝ նկարներ, գին,
բնութագրեր, հատակագիծ եւ այլ ինֆորմացիա։
```
```
 Տվալների սինխրոն թարմացում
Քարտում կատարված ցանկացած փոփոխություն (նոր նկարներ, գնի փոփոխություն,
կարգաիճակ եւ այլն) ավտոմատ թարմացվում է նաեւ լենդինգում։
```
```
 Անհատական պաշտպանված հղում
Յուրաքանյուր սեփականատիրոջ համար ստեղծվում է եզակի հղում, որի իջոցով նա տեսնում է
իայն իր բնակարանի տվալները։
```
```
 Գործընթացների եւ կարգաիճակների տեսանեիություն
Լենդինգում սեփականատերը տեսնում է գործարքի ընթացքը, կարգաիճակները եւ բոլոր
թարմացումները իրական ժամանակում։
```
## 📌 Ֆունկցիոնալ 2 — Apartments բաժնի ներքին թարմացում

```
 Ցուցադրման նոր ձեւաչափի ավելացում
Գործող երկու ձեւաչափերին (ցանկային եւ քարտային) ավելացվում է երրորդ՝ նոր
կառուցվածքային ձեւաչափ։
```
```
 Հատակային (Floor View) ցուցադրում
Տվալները ներկայացվում են ըստ հարկերի՝ շենքի կառուցվածքին համապատասխան։
```
```
 Հարկի պլանի ցուցադրում
Յուրաքանյուր հարկի բլոկում վերեւում ցուցադրվում է տվալ հարկի հատակագիծը։
```
```
 Բնակարանների խմբավորում ըստ հարկի
Հատակագծի ներքեւում ցուցադրվում են տվալ հարկին պատկանող բոլոր բնակարանները։
```
```
 Հերթական դիտում ըստ հարկերի
Օգտատերը սկրոի իջոցով հաջորդաբար դիտում է շենքի բոլոր հարկերը՝ վերեւից ներքեւ
տրամաբանական կառուցվածքո։
```
```
 Բնակարաններում տեքստային դաշտերի ավելացում
Յուրաքանյուր բնակարանի քարտում ավելացվում են տեքստային դաշտեր՝ տվալ բնակարանին
```
