# Ներմուծման ֆայլերի լրացման հրահանգ

Լրացրեք երեք CSV ֆայլ և վերադարձրեք դրանք բազա ներմուծելու համար։ **Այս ներմուծման մեջ ֆայլեր (կցված ֆայլեր, փաստաթղթեր) չեն փոխանցվում** — միայն թաղամասերի, շենքերի և բնակարանների տվյալները։

---

## Լրացման կարգը

1. **districts_template.csv** — նախ թաղամասերը  
2. **buildings_template.csv** — ապա շենքերը (նշեք առաջին ֆայլի թաղամասի slug-ը)  
3. **apartments_template.csv** — ապա բնակարանները (նշեք թաղամասի slug-ը և շենքի slug-ը)

Վերնագրերի (առաջին) տողը մի ջնջեք։ Օրինակի տողերը կարող եք ջնջել կամ թողնել որպես նմուշ։

---

## 1. Թաղամասեր (districts_template.csv)

| Սյունակ | Պարտադիր | Նկարագրություն |
|---------|----------|-----------------|
| name    | Այո       | Թաղամասի անվանում (օրինակ՝ Kentron, Arabkir) |
| slug    | Այո       | Եզակի լատինական նույնականացուցիչ՝ միայն փոքրատառ տառեր, թվեր և գծիկներ (օրինակ՝ kentron, arabkir) |

---

## 2. Շենքեր (buildings_template.csv)

| Սյունակ       | Պարտադիր | Նկարագրություն |
|---------------|----------|-----------------|
| district_slug | Այո       | Թաղամասի slug-ը թաղամասերի ֆայլից (օրինակ՝ kentron) |
| name          | Այո       | Շենքի անվանում (օրինակ՝ Tower 1) |
| slug          | Այո       | Շենքի եզակի նույնականացուցիչ թաղամասի ներսում (օրինակ՝ tower-1, block-a) |

---

## 3. Բնակարաններ (apartments_template.csv)

**Պարտադիր սյունակներ.**

| Սյունակ        | Նկարագրություն |
|----------------|-----------------|
| district_slug  | Թաղամասի slug (ինչպես թաղամասերի ֆայլում) |
| building_slug  | Շենքի slug (ինչպես շենքերի ֆայլում) |
| apartment_no   | Բնակարանի համար (օրինակ՝ 12-05, 3-11) |
| status         | Մեկը հետևյալից՝ **UPCOMING**, **AVAILABLE**, **RESERVED**, **SOLD** |
| sales_type     | Մեկը հետևյալից՝ **UNSOLD**, **MORTGAGE**, **CASH**, **TIMEBASED** |

**Մնացած սյունակները կարող եք դատարկ թողնել**, եթե տվյալներ չկան.

| Սյունակ                | Նկարագրություն |
|------------------------|-----------------|
| apartment_type         | Թիվ (բնակարանի տիպ) |
| sqm                    | Մակերես, մ² (թիվ) |
| price_sqm              | Գին մ²-ի համար, AMD (թիվ) |
| total_price            | Ընդհանուր գին, AMD (թիվ) |
| total_paid             | Արդեն վճարված, AMD (թիվ) |
| deal_date              | Գործարքի ամսաթիվ (ձևաչափ՝ YYYY-MM-DD, օրինակ՝ 2026-01-20) |
| ownership_name         | Սեփականատիրոջ անուն ազգանուն |
| email                  | Email |
| passport_tax_no        | Անձնագիր / հարկային համար |
| phone                  | Հեռախոս |
| deal_description       | Գործարքի նկարագրություն (մինչև 500 նիշ) |
| matter_link            | Հղում (URL) |
| floorplan_distribution | Պլանավորման տեքստ (մինչև 500 նիշ) |
| exterior_link         | Հղում (URL) |
| exterior_link2         | Հղում (URL) |

Դատարկ բջիջները նորմալ են, ամեն ինչ լրացնելու կարիք չկա։

---

## Ինչը չենք լրացնում

- **ID** (id) — ներմուծման ժամանակ ավտոմատ ստեղծվում են։  
- **Ամսաթվեր** created_at, updated_at — ներմուծման ժամանակ դրվում են։  
- **Ֆայլեր և կցված ֆայլեր** (փաստաթղթեր, պլաններ, նկարներ) — այս ներմուծման մեջ չեն փոխանցվում, դրանք կարող են ավելացվել ավելի ուշ համակարգում։

---

## Ֆայլերի ձևաչափ

- Կոդավորում՝ **UTF-8** (Excel-ում պահպանելիս ընտրեք «CSV UTF-8» կամ «Unicode (UTF-8)»)։  
- Բաժանարար՝ **ստորակետ** (,)։  
- Եթե բջջում կա ստորակետ կամ տողի անցում — արժեքը դրեք չակերտների մեջ, օրինակ՝ `"Տեքստ, ստորակետով"`։

Լրացնելուց հետո վերադարձրեք երեք ֆայլ։ Ներմուծելու համար դրեք դրանք `data/import/` պանակում հետևյալ անուններով.
- **districts.csv**
- **buildings.csv**
- **apartments.csv**

Ներմուծման գործարկում (նախագծի արմատից).
```bash
npm run db:import-csv
```
Սկրիպտը կվերցնի նաև `*_template.csv` ֆայլերը, եթե առանց վերջածանցի ֆայլեր չկան։
